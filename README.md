## Бот учета посещаемости лекций

Телеграм-бот для учета посещаемости лекций студентами.

### Возможности

- Регистрация студентов (ФИО и номер группы).
- Разделение ролей: **Студент** и **Администратор**.
- Администраторы создают лекции (название, дата/время, здание).
- Студенты отмечаются на лекции через отправку геолокации.
- Проверка, что отметка:
  - сделана **во время проведения лекции**;
  - находится **в радиусе заданного здания**;
  - использует свежую геолокацию с приемлемой точностью.
- Экспорт статистики посещаемости в CSV (ФИО и группа студентов).

### Установка

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Установите переменные окружения:

- `TELEGRAM_BOT_TOKEN` — токен бота.
- `ADMIN_IDS` — список Telegram ID администраторов через запятую, например: `123456789,987654321`.

В PowerShell:

```powershell
$env:TELEGRAM_BOT_TOKEN = "ВАШ_ТОКЕН"
$env:ADMIN_IDS = "12345"
python almazov_lectures_bot.py
```

### Запуск бота

```bash
python almazov_lectures_bot.py
```

### Замечания по проверке геолокации

- Рекомендуется отправлять **live location** (живую геолокацию) в Telegram.
- Бот отклоняет локацию:
  - с низкой точностью (слишком большой `horizontal_accuracy`);
  - отправленную задолго до момента проверки;
  - находящуюся за пределами допустимого радиуса от здания.
